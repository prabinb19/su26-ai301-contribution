## How this tracker is organized

Each contribution gets its own log file (e.g. `contribution_readme1.md`) following the
program's phased structure:

- **Phase I — Issue Selection:** pick an issue, summarize the problem, explain why.
- **Phase II — Reproduce & Plan:** set up the project locally, reproduce the bug, draft an approach.
- **Phase III — Implement & Test:** build the fix and write tests.
- **Phase IV — Pull Request:** submit the PR and track maintainer feedback through to merge.
